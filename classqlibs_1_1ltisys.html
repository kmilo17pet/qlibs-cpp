◆ excite()

real_t ltisys::excite ( real_t u)
noexcept

Drives the LTI system recursively using the provided input sample.

This function evaluates the system response based on the given input signal and the internal state of the system. It must be called periodically with a fixed time step to maintain correct system behavior.

Precondition
The instance must be properly initialized before calling this method.
Note
The user is responsible for ensuring that this function is invoked at consistent intervals equal to the system's time step dt. This timing should be enforced using a hardware timer, software timer, real-time task, or another reliable timing mechanism.
Parameters
[in]uA new input sample that excites (drives) the system.
Returns
The system's output (response) at the current time step.
